# Railway Configuration for ERP System

This guide helps you deploy the ERP System to Railway with PostgreSQL.

## Prerequisites

- Railway account (https://railway.app)
- GitHub account with project repository
- Basic knowledge of command line

## Step-by-Step Deployment

### 1. Create Railway Project

1. Visit [Railway Dashboard](https://railway.app/dashboard)
2. Click "New Project"
3. Select "Deploy from GitHub"
4. Authorize Railway to access your GitHub
5. Select your ERP_System repository

### 2. Add PostgreSQL Database

1. In your Railway project, click "Add Service"
2. Select "PostgreSQL" from the marketplace
3. Wait for the database to initialize
4. The `DATABASE_URL` will be automatically added to your environment

### 3. Configure Environment Variables

In Railway project settings, add these environment variables:

```
# Django Settings
DEBUG=False
SECRET_KEY=your-production-secret-key-here
ALLOWED_HOSTS=your-project.railway.app,your-domain.com

# Database (Railway PostgreSQL auto-generates DATABASE_URL)
# DATABASE_URL=postgresql://... (automatically provided)

# Optional: Custom settings
LANGUAGE_CODE=en-us
TIME_ZONE=UTC
```

### 4. Create Procfile

The project already includes a `Procfile`:

```
web: gunicorn erp_system.wsgi:application --bind 0.0.0.0:$PORT
```

This tells Railway how to run your application.

### 5. Generate Secret Key

Generate a secure SECRET_KEY:

```python
from django.core.management.utils import get_random_secret_key
print(get_random_secret_key())
```

Copy this value and paste it into the `SECRET_KEY` environment variable.

### 6. Deploy

1. Push your changes to GitHub:
```bash
git add .
git commit -m "Deploy to Railway"
git push origin main
```

2. Railway automatically detects changes and redeploys
3. Check deployment status in Railway dashboard
4. Wait for the build and deployment to complete (typically 2-5 minutes)

### 7. Run Initial Setup

Once deployment is complete:

```bash
# In Railway project terminal or via CLI:
railway run python manage.py migrate
railway run python manage.py createsuperuser
```

Or use Railway CLI:

```bash
# Install Railway CLI
npm install -g @railway/cli

# Login
railway login

# Run commands
railway run python manage.py migrate
railway run python manage.py createsuperuser
```

### 8. Verify Deployment

1. Visit your Railway project URL
2. Access login page
3. Create a test account
4. Login and verify functionality

### Required

| Variable | Description | Example |
|----------|-------------|---------|
| `SECRET_KEY` | Django secret key | (generate random) |
| `DEBUG` | Debug mode (False for production) | False |
| `ALLOWED_HOSTS` | Allowed domain names | your-app.railway.app |

### Auto-Generated by Railway

| Variable | Source |
|----------|--------|
| `DATABASE_URL` | PostgreSQL service |
| `PORT` | Railway platform |

### Optional

| Variable | Default | Description |
|----------|---------|-------------|
| `LANGUAGE_CODE` | en-us | Language setting |
| `TIME_ZONE` | UTC | Timezone |

## Database Connection

Railway PostgreSQL provides `DATABASE_URL` automatically. The Django settings.py handles:

1. `DATABASE_URL` - Parse PostgreSQL connection
2. Individual `PG*` variables - Alternative connection
3. SQLite fallback - For local development

## Static Files

Static files are handled via `collectstatic`:

1. Railway runs `python manage.py collectstatic --noinput`
2. Files are served from `/staticfiles/` directory
3. Configure CDN in Django settings if needed

## Logs and Debugging

### View Logs

In Railway dashboard:
1. Select your project
2. Click on the web service
3. View "Logs" tab in real-time

### Common Issues

**Issue**: `ModuleNotFoundError: No module named 'x'`
- **Solution**: Ensure all packages are in `requirements.txt`

**Issue**: `django.core.exceptions.ImproperlyConfigured: Missing DATABASE_URL`
- **Solution**: Verify PostgreSQL service is linked to web service

**Issue**: `PermissionError: [Errno 13] Permission denied`
- **Solution**: Check file permissions and directory ownership

**Issue**: `DisallowedHost exception`
- **Solution**: Add your domain to `ALLOWED_HOSTS` environment variable

## Performance Optimization

### 1. Database Connection Pooling

Already configured in settings.py:
```python
'CONN_MAX_AGE': 600  # 10 minutes
```

### 2. Static Files Cache

Configure in Railway environment:
```
STATIC_ROOT=/app/staticfiles
STATIC_URL=/static/
```

### 3. Database Query Optimization

Use Django ORM select_related and prefetch_related:
```python
students = Student.objects.select_related('user').all()
```

## Scaling

### Vertical Scaling (increase size)
1. Go to Railway project settings
2. Adjust "Memory" and "CPU" sliders
3. Changes apply to new deployments

### Horizontal Scaling (multiple instances)
1. Open project settings
2. Set "Replicas" > 1
3. Railway load balances automatically

## Backups

Railway PostgreSQL provides automatic daily backups. To restore:

1. Go to PostgreSQL service
2. Click "Backups" tab
3. Select backup date
4. Click "Restore"

For manual backups:

```bash
# Export database
railway run pg_dump $DATABASE_URL > backup.sql

# Restore database
railway run psql $DATABASE_URL < backup.sql
```

## Domain Setup

### Custom Domain

1. In Railway project, go to "Settings"
2. Add your domain to `ALLOWED_HOSTS`
3. Point your domain DNS to Railway:
   - Type: `CNAME`
   - Value: Your Railway app URL

### SSL Certificate

Railway provides free SSL certificates automatically via Let's Encrypt.

## Monitoring

```python
def health_check(request):

```

Configure in Railway health check settings.

### Alerts

## Advanced Configuration

```python
# In settings.py
if DEBUG:
    # Development settings

    # Production settings
    SECURE_SSL_REDIRECT = True
    SESSION_COOKIE_SECURE = True
    CSRF_COOKIE_SECURE = True
```

### Custom Domain with Subdomain

```

    'yourdomain.com',
    'www.yourdomain.com',
    'api.yourdomain.com',
    'your-app.railway.app'
]
```
